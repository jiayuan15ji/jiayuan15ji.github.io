设计模式1-策略模式与观察者模式

发表于 2017-03-01

策略模式

顾名思义，对策略的封装。一类东西有共同的特点，都可以执行相同的动作，然而动作的细节却不尽相同，这个时候便将这个动作提取出来，作为一个接口，具体的动作实现这个接口，再与这类东西的超类进行组合。子类只需在构造时，设置好这些动作的具体实现类即可。当我们需要新增具有新动作的子类时，我们只需要新增动作的实现，而不需要修改超类。符合开闭原则，即对新增开放，对修改关闭。

定义算法族，分别封装起来，让它们之间可以相互替换，此模式让算法的变化独立于使用算法的客户。

观察者模式

观察者模式分为两类角色，一方是观察的人，另一方是被观察的人。被观察的人我们叫做主题好了。为什么要观察，因为主题有些东西或者说状态的变化是观察的人需要知道的。那么我们就需要在主题发生变化时，通知观察者这些变化，然后观察者做出自己的反应。首先，我们需要一个主题接口，定义三个方法：添加观察者，删除观察者，通知观察者。主题具体的实现类则需要组合的方式，对观察者的队列进行操作。关键还要有一个变化发生时调用的方法，以遍历整个观察者队列进行推送后的观察者自己的处理。因此，观察者就需要一个接口，定义一个处理的方法。具体的观察者就根据自己情况进行不同的处理。这样做使得主题不需要关心观察者的实现是否发生变化，或者是否出现了新的观察者，只需要知道，他们共同实现了观察者接口，所以观察者的变化不会影响到主题的代码，松耦合。

另外观察者模式还分为 推 和 拉 两种方式。

Java API中自带了主题超类 Observable 和观察者接口 Observer 。以继承的方式存在缺陷需要注意。

在对象之间定义一对多的依赖，这样一来，当一个对象改变状态，以来它的对象都会收到通知，并自动更新。
